Namespaces
Namespaces
NamespaceDescription
Assets.BlobDistributors
Contains the logic by which blobs are distributed from BlobSites.
Assets.Blobs
Contains the logic for the ResourceBlobs themselves, including the available ResourceTypes.
Assets.BlobSites
Contains the logic for storing and placing blobs in various locations. Every MapNode contains a BlobSite that manages its contents, placement permissions, extraction permissions, and capacities.
Assets.ConstructionZones
Contains the logic that handles ConstructionZones and the ConstructionProjects that control them. Elements in this namespace define the ways in which players can construct Societies, ResourceDepots, HighwayManagers, and clear and plant forests.
Assets.Core
Contains two types of classes: those that control the simulation (centered around SimulationControl) and those that control the UI (centered around UIControl). The classes in this module are designed to create a very thick layer of abstraction between player input/output and the code of the underlying simulation. Modules in Core are supposed to separate the UI code from any of the implementation details specified in other namespaces. Whether this was a beneficial decision remains to be seen.
Assets.HighwayManager
Defines the logic for the HighwayManager object, a gameplay element that the players can use to turn resources into highway throughput.
Assets.Highways
Contains the logic by which ResourceBlobs are moved between nodes on the map. Highways consist of a pair of BlobTubes, and have commands for pulling resources from and pushing resources into the BlobSites at their endpoints.
Assets.Map
Contains the logic for the construction and maintenance of the map. This includes MapGraph, MapNode, and MapEdge, along with some graph traversal algorithms.
Assets.ResourceDepots
Defines the logic for the ResourceDepot gameplay element, which is a place where resources can be stored and extracted with ease.
Assets.Scoring
Contains the logic for scoring a given map configuration, managing victory conditions, and flagging when victory should occur.
Assets.Session
Contains all the logic associated with session, serializiable records of the state of a given map. Maps and saved games are stored as sessions, which are exchanged between persistent storage and the current runtime via the SessionManager class. This namespace also contains interaction with the file system through FileSystemLiasion.
Assets.Societies
Contains the logic that manages societies, by far the most complex gameplay elements in the game.
Assets.UI
Contains numerous classes for all of the UI in the game, organized loosely by the non-UI modules they act as UI for. The classes in this namespace are largely servants to UIControl and its subservient modules, found in Assets.Core. These classes tend to be the most user-facing elements and the most subject to change. For this reason, this namespace is largely not covered by unit tests like the more simulation-focused namespaces are.
Assets.UI.Blobs
Contains UI elements associated with resource blobs.
Assets.UI.ConstructionZones
Contains UI elements associated with construction zones and construction projects.
Assets.UI.EscapeMenu
Contains UI elements for the Escape Screen, or the screen brought up while the game is running by pressing Escape. It defines several subscreens for saving and loading sessions as well as exiting the current session.
Assets.UI.HighwayManager
Contains UI elements associated with highway managers.
Assets.UI.Highways
Contains UI elements associated with highways.
Assets.UI.ResourceDepots
Contains UI elements associated with resource depots.
Assets.UI.Scoring
Contains UI elements associated with scoring, including victory.
Assets.UI.Session
Contains UI elements associated with serializable sessions.
Assets.UI.Societies
Contains UI elements associated with societies.
Assets.UI.TitleScreen
Contains all of the UI elements that manage everything that happens outside of the main execution of the game. This includes starting new games, loading saved sessions, the credits and attribution page, as well as a brief description of how to play the game.
Assets.Util
A small namespace containing a handful of utilities that did not fit cleanly into another category.
UnityCustomUtilities.Grids
Contains classes for a hexagonal grid of arbitrary tiles. This implementation is based off a detailed blog from Red Blob Games (http://www.redblobgames.com/grids/hexagons/), which was also source of much of the code here, though it's been modified since then.
