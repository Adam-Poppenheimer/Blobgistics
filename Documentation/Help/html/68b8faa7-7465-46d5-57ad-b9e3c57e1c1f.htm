Remarks
Intelligent panels are primarily used in this codebase to display information about a selected object. It makes use of the selection semantics that Unity offers. However, selection changes every time any selectable object is selected or deselected, which may include UI elements like buttons that are part of this panel. We solve this problem by informing the panel when its children have been selected and deselected, as well. The panel should only be deactivated if it or one of its children has been deselected and neither it nor one of its children has been reselected. Every child object with a selectable component must call into these two events via an EventTrigger in order to handle panel selection behavior properly.
