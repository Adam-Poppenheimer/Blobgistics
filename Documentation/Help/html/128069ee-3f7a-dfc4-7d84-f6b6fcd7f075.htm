Remarks
While this class contains a lot of code, most of its complexity boils down to carefully recording and then reconstructing various objects. Order is somewhat important here, as is generating a mapping between the old IDs of various gameplay elements and their new IDs (since the IDs are liable to change when the new, equivalent object is constructed). This elaborate process is necessary in order to allow serialization during gameplay, but has also been repurposed to save maps at design time for the sake of simplicity.
